A simple guide to DUST-20 on Bitcoin

Understand DUST-20 before you send.

DUST-20 is an experimental way to represent tokens on Bitcoin. Each unit is tied to a fixed number of satoshis, the smallest parts of bitcoin. Start small, check every transaction, and only approve what you understand.

!

DUST-20 is still experimental

There is no single final DUST-20 rulebook yet. A Bitcoin transaction can confirm while different DUST-20 tools show it differently. Start with a small test, wait for confirmation, and never send more than you can afford to lose.

Start here

Four things to know before you approve.

You do not need to learn every technical word. These four checks help you make a safer decision when DUST-20 appears in a wallet or app.

01 / SATS

Each unit has bitcoin backing

A satoshi, or sat, is the smallest part of bitcoin. DUST-20 uses sats as the backing for token units.

02 / OUTPUT

Your DUST-20 has a location

DUST-20 is attached to a specific piece of a Bitcoin transaction, called an output. That location matters when you send it.

03 / CHANGE

Keep what you do not send

If you send part of a larger DUST-20 amount, the rest needs its own change output back to you.

04 / FEE

Pay fees with regular bitcoin

The Bitcoin network fee should come from separate regular bitcoin, not from the sats that back your DUST-20.

How it works

Set a rule, create units, then move them carefully.

The technical details live on the later pages. This is the everyday version: keep the token amount, its Bitcoin backing, and the network fee clearly separated.

Set the rule

A deploy describes how many sats back one unit and how much can exist.

Create an amount

A mint creates units whose stated backing matches the actual Bitcoin output.

Send with change

A partial send needs a recipient output and a separate DUST-20 change output for the rest.

Check after confirmation

Make sure the recipient, your remaining amount, and the balance view all agree.
