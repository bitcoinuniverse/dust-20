Bitcoin UTXO field guide

DUST-20, made legible.

DUST-20 is an experimental fungibility model where token units are represented by satoshis in colored UTXOs. This site explains what the legacy guide says, what Universe tooling currently emits, and what remains unresolved.

Experimental profile, not Bitcoin consensus

DUST-20 is an experiment. A valid-looking JSON payload, wallet screen, marketplace listing, or indexer row does not prove a safe allocation transition. Review the transaction outputs and use only low-value, controlled tests until the unresolved rules are settled.

One model, three layers

Know which statement you are relying on.

The old guide, the live application, and a robust indexer do not yet describe one fully settled contract. This documentation keeps their boundaries visible.

01 / LEGACY GUIDE

Documented payload facts

The legacy DUST-20 guide documents deploy and mint fields, the satoshi arithmetic, and a colored-change transfer example. It does not define every transaction or indexer edge case.

02 / UNIVERSE PROFILE

Implementation behavior

Universe Inscribe currently constructs deploy, mint, and transfer previews. Its payload choices can differ from the legacy example, so they are identified as implementation behavior, not silent consensus.

03 / OPEN RULES

What must not be guessed

Duplicate tickers, output ordering, multiple colored inputs, RBF, reorgs, accidental burns, and indexer disagreement need explicit policy before balances or automation can be treated as dependable.

The lifecycle

Metadata starts the story. Outputs carry it forward.

Every step should be observable to a wallet, a user, and an indexer. The wallet must never let a fee quietly consume the satoshi backing that represents a token allocation.

Deploy

Declare a ticker, supply, explicit unit size, and a maximum backing calculation in the legacy profile.

Mint

Inscribe an amount whose declared `sats` and actual inscription output value both equal the exact allocation.

Transfer

Spend colored inputs to recipient allocation outputs and an explicit colored change output when a remainder exists.

Reconcile

Confirm, then compare on-chain outputs with the indexer. Treat a mismatch as an investigation, not a cosmetic display problem.
