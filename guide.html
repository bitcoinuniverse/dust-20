Build and verify

Do not make the wallet the first review.

A DUST-20 transaction is ready to sign only when its payload, selected UTXOs, output values, recipient, colored change, fee funding, and indexer interpretation agree. Build that proof before a wallet prompt appears.

PreflightDeployMintTransferErrors

Preflight

Collect evidence before building a transaction.

A transaction builder should keep protocol-critical values visible in the application UI, not hide them behind generic send language or a wallet-specific confirmation screen.

NETWORK

Confirm the environment

Show the Bitcoin network, active wallet, payment address, ordinal or receiver address, and fee rate. Stop on a network or address-type mismatch.

STATE

Refresh external state

Read the deployment, selected output status, and listing state from the configured indexer. Preserve the retrieval time and source with the request.

MATH

Calculate with integers

Use exact whole-satoshi and whole-unit arithmetic. Reject blank, negative, fractional, overflowed, or rounded values before a PSBT is constructed.

Deploy

Establish the ratio explicitly.

Deployment starts an interpretation rule. It should be easy for an independent reader to reconstruct the unit size and total promised backing without relying on UI defaults.

  1. Require a nonempty ticker and show the exact original spelling that will be inscribed. The legacy guide does not settle case normalization or duplicate resolution.
  2. Require positive integer `supply` and `unit_sats`, then compute `max_sats = supply * unit_sats` exactly.
  3. Show optional `lim_sats` separately and explain that it caps one mint's satoshi backing only where the relevant policy applies.
  4. Build the deploy text from the reviewed values. Never manually concatenate unvalidated user input into JSON.
  5. After broadcast, record the transaction ID, output, confirmation state, and observed indexer interpretation. A submitted deploy is not final proof of unique ticker ownership.

Mint

Bind amount, declared sats, and output sats.

The most important mint check is triplicate: the deployment's unit size, the JSON `sats` field, and the actual inscription output must all report the same exact allocation.

mint_sats = amt * unit_satsWith 91 units and 546 sats per unit, both the payload and inscription output must be 49,686 sats.

Mint review panel

Display ticker, deployment output, amount, unit size, computed sats, declared sats, actual output value, remaining supply, optional limit, fee rate, receiver, and funding UTXOs in one stable review surface.

  • Resolve an accepted deployment before accepting `amt`.
  • Reject if `sats != amt * unit_sats` or if actual output value differs from `sats`.
  • Reject if `sats > lim_sats` where a deployment or indexer has a limit.
  • Reject if prior minted units plus `amt` exceed supply.
  • Wait for confirmation and reconciliation before presenting the new output as an available balance.

Transfer and change

A partial send needs an explicit colored remainder.

The legacy guide offers a split example but not a normative transfer JSON schema. Use this as conservative transaction-construction guidance, not a claim of consensus for every multi-input case.

Colored input49,686 sats91 units at 546 sats each
→
Recipient colored output5,460 sats10 units to recipient
Colored change output44,226 sats81 units back to sender
COLORED SIDE

Conserve allocation sats

For a normal transfer, sum of selected colored input sats should equal recipient colored sats plus colored change sats. Every claimed colored output should be divisible by the deployment unit size.

CARDINAL SIDE

Fund the miner fee separately

Use a non-colored cardinal UTXO for network fees and ordinary change. Do not trim colored backing to pay a fee and then call the resulting output a valid token allocation.

Failure behavior

Stop safely, explain specifically.

Do not turn an unavailable indexer, an ambiguous broadcast, or a changed listing into an empty successful state. A user needs enough information to inspect, retry safely, or seek support.
